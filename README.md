# MULoop

📝 アプリケーション開発計画書 兼 設計仕様
書
1. 企画概要
●
●
●
●
アプリ名： MULoop（ミューループ）
ターゲット： 同じ大学のキャンパスに通う大学生
コア・コンセプト： 「空きコマを、いちばん楽しい時間に。」
目的・価値： 大学生活特有の「空きコマ」や「突発的な暇な時間」をビジュアルで共有し、
学内の偶発的なコミュニケーションやサークル・イベントへの参加を活発化させる。
2. コア機能と「インスタ」との差別化
パブリック（イベント・未来）とプライベート（今ココ・日常）を融合させたハイブリッド型設計。
●
●
① タイムライン型時間割（パブリック機能）
○
1限〜5限のグリッド形式のカレンダー
。
○
大学全体のサークル行事、ゼミの発表会、テスト勉強会などの「企画」が時間割
の枠内にオープンに表示される。
○
【インスタとの違い】 自分の空きコマの時間に開催されているイベントが自動で
フィルタリングされ、ひと目でわかる。
② 「今ココ！」キャンパス・スナップ（プライベート機能）
○
「今、学食にいる」「ラウンジで課題中」などの日常を、写真＋場所タグ＋ステータ
スで投稿（24時間で消える）。
○
繋がる範囲を「友達・サークル内」に限定することで、防犯面やプライバシーの安
心感を担保。
○
【インスタとの違い】 「同じ大学の、同じキャンパスに今いる人」しか見ないため、
内輪ノリや防犯を気にせず、地味でリアルな日常を気軽に投稿できる。

4. 開発環境・技術スタック
ゼミのサーバーへのデプロイ（Webアプリケーション）を前提としたモダンな構成。
●
●
●
●
フロントエンド（画面側）： React (JavaScript / TypeScript)
バックエンド（サーバー・データ側）： PHP（生PHP または Laravelなどのフレームワーク）
UIコンポーネントライブラリ： Yamada UI
○
選定理由：国産で完全日本語ドキュメントのため開発効率が高い。Grid コン
ポーネントが時間割のマス目UIを実装するのに最適。
デザイン・設計ツール： Figma（Yamada UIのFigma Kitを活用）
5. 開発ロードマップ（全4ステップ）
🏁 STEP 1：画面設計とデータベース設計（期間：1〜2週間）
●
やること：
○
本仕様書のワイヤーフレームを元に、必要なデータ（ユーザー情報、投稿写真の
URL、場所タグ、時間割のコマ情報など）のテーブル設計をノートに書き出す。
🎨 STEP 2：Figmaでのデザイン清書（期間：2週間）
●
やること：
○
Figmaを立ち上げ、Yamada UIが公式に提供している「Figma UI Kit」のコンポー
ネント（ボタンやフォームなど）を活用して、実際のWeb画面の完成見本を作成す
る。
⚙ STEP 3：React ✕ PHP 実装（期間：1〜1.5ヶ月）
●
やること：
○
○
バックエンド： PHPでデータベースと接続し、画像のアップロード機能や時間割
データの読み書きAPIを作成する。
フロントエンド： ReactとYamada UIを導入し、STEP 2で作ったデザインをコード
に起こす。PHPのAPIと連携させて、データが画面に反映されるようにする。
🚀 STEP 4：ゼミサーバーへのデプロイとテスト（期間：2週間）
●
やること：
○
○
成果物をゼミのサーバーにアップロード（デプロイ）。
ゼミ生全員にスマホのブラウザから触ってもらい、バグの修正や「本当に空きコマ
に集まりたくなるか」の検証を行い、ゼミの最終発表スライドをまとめる。
